ATTRIBUTI


Gli attributi specificano una funzione o una tipologia dell'elemento o del tag che stiamo usando.
Si "racchiudono" sempre una in una coppia definita chiave-valore, dove la chiave si riferisce all'attributo mentre il valore è cio a che viene specificato dallattributo.
Solitamente un determinato tag e attributo(chiave-valore) vengono messi assieme proprio perchè nascono a "coppie".
Inoltre molti attributi possono essere utlizzati con più tag, con invece insolita per i tag che difficilmente si scambiano gli attributi anche se molti tag possono essere accompiati con un sacco di attributi

IMPUT tag (è un tag particolare)


Serve a far interagire l'utente con il form tag legato al database in back-end (per questo presenta valori dell'attributo diversi).
In pratica tutti gli input (appunto perchè il tag input è uno solo ma si differiscono tra loro per il valore dell'attributo) con i loro attributo e specifico valore contenuti e il form col suo attributo e valore, creano una struttura organizzata che poi si collegherà al database in back-end.
Tag input e attributo type vanno praticamente sempre assieme, cambiano invece i valori (input type="...")
L'input tag non ha bisogno di chiusura

FILE PATH


Serve a inserire le immagini come valore dell'src, perchè in teoria non andrebbe messo il link perchè se non esiste un immagine in internet come si fa se non ha un URL
Esistono 4 tipi di file path
  • nome dell'immagine.jpg: nel caso l'immagine e il file html si trovino nella stessa cartella
  • nome della cartella dell'immagine/nome dell'immagine.jpg: nel caso l'immgine e il file html si trovino in due cartelle diverse
  • /nome della cartella dell'immagine/nome dell'immagine.jpg: non ho capito
  • nome della cartella del file html/nome dell'immagine.jpg: se l'immagine si trova in una cartella che sta al di sopra (proprio come posizione) della cartella del file html
